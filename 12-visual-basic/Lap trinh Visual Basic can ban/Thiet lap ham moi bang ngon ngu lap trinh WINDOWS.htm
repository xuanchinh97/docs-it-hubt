(Visual Basic for Application)  

Visual Basic cãn bản

Ngôn ngữ lập trình Windows Visual Basic là một trong những ngôn ngữ hiện ðại nhất hiện nay. Ðặc tính cõ bản là hýớng ðối týợng (Visual) nghĩa là ngýời lập trình chỉ việc thao tác muốn công việc diễn ra nhý thế nào, Visual Basic sẽ tự viết cho ta phần lập trình.

Visual Basic ðýợc bắt ðầu hình thành và phát triển từ Windows 3.XX chủ yếu là hỗ trợ chýõng trình EXCEL. Các chýõng trình khác chạy trong Windows nhý WORD, ACCESS lại có ngôn ngữ riêng nhý WORD BASIC, ACCESS BASIC... nói chung là họ Basic trong Windows. Ngoài ra Microsoft còn cung cấp thêm bộ chýõng trình Visual Basic riêng ðể lập trình móc nối. Khai thác Visual Basic sẽ khám phá ra rất nhiều những ðặc tính tuyệt vời. Và càng tuyệt vời hõn khi Windows 95 và Windows 98 ðã ðýợc Microsoft hỗ trợ Visual Basic trong toàn bộ các ứng dụng Windows: WINWORD, EXCEL, ACCESS. Ta ðều có thể lập trình bằng Visual Basic thiết kế hàm và lệnh con mới cho chính cả 3 môi trýờng trên.

Có nhiều cách ðể ðạt mục ðích, tất nhiên làm tất cả mọi việc riêng cho mình từ ðầu ðến cuối cũng là ðiều tốt. Nhýng trýớc mắt có lẽ nhanh chóng và hợp lý hõn cả là ta tận dụng ngay các chýõng trình và thuật toán hiện ðại nhất thế giới của Microsoft trong EXCEL ðể phục vụ cho công việc của mình, ðồng thời có thể góp phần xây dựng bổ sung thêm những gì cho mình bằng chính Visual Basic.

Bạn có thể tham khảo các tài liệu về Visual Basic, phần dýới ðây ðýợc lýợc soạn từ bản dịch Visual Basic For Application của Trung tâm Tin học ABC. Bạn có thể ðọc chi tiết ở những trang chi tiết.

*Ðể thiết lập hàm mới bằng Visual Basic ta làm các býớc sau:

Gõ Alt + F11. Insert   Module  (Mô ðun)

Hiện ra cửa sổ soạn thảo ðể ta ðánh.

*Dòng 1 nếu không có ta ðánh vào dòng:

Option Explicit

Dùng ðể báo cho Visual Basic biết là các biến sử dụng phải ðýợc khai báo.

Các dòng tiếp theo:

FUNCTION Tên_hàm (Danh_sách_các_biến_truyền As Kiểu_biến) As Kiểu_biến_trả_về_của_hàm.

DIM Tên_biến_ghi_kết_quả (As Kiểu_biến)

DIM Tên_các_biến_sử_dụng (As Kiểu_biến)

Nói riêng về biến:

Biến là một vùng các ô nhớ trong RAM  ta phải ðặt tên ðể tránh nhầm lẫn khi dùng ðến nó. Tại một thời ðiểm trong biến chỉ có một giá trị, nếu biến không ðýợc truyền vào thì giá trị của biến ban ðầu là không có gì hoặc là 0.

Trong quá trình lập trình muốn ðặt vào biến một giá trị ta dùng công thức:

Tên_biến = Giá trị cần ðặt.

Ví dụ: Ðặt biến NHIETDO bằng 100 ðộ. Thì dùng:

NHIETDO = 100

Muốn thêm giá trị vào biến thì ta dùng công thức:

Tên_biến = Tên_biến + Giá_trị_thêm

Ví dụ: Tãng biến NHIETDO thêm 50 ðộ nữa.

NHIETDO = NHIETDO +50

Có thể sử dụng phýõng pháp ðặt giá trị mới của biến này thông qua một biến khác bằng cách sử dụng công thức:

Biến_cần_ðặt_mới = Biến_cãn_cứ  (phép_tính với biến)

Ví dụ ðặt biến NHIETDO luôn lớn hõn biến NHIETDO_NGOAITROI 10 ðộ.

NHIETDO = NHIETDO_NGOAITROI + 10

Lúc này giá trị của biến ðã là giá trị mới. Giá trị cũ của biến bị xoá ði.

Nếu muốn ghi lại giá trị cũ ta phải dùng một biến khác ðể ghi lại, sau ðó ðến khi nào cần sử dụng giá trị cũ thì trả lại

Ví dụ: Ðýa vào biến ghi lại giá trị của biến nhiệt ðộ lúc 10 giờ.

LUC10GIO = NHIETDO

.......

NHIETDO = LUC10GIO

Bình thýờng ta không cần ðánh thêm as Kiểu biến khi ðó tốc ðộ xử lý của máy sẽ chậm nếu những hàm phức tạp. Ta ðánh thêm as Kiểu biến ðể tãng tốc ðộ xử lý.

- Nếu biến là các số nguyên chọn Kiểu_biến là Integer

- Nếu biến là chuỗi ký tự chọn Kiểu_biến là String

- Nếu biến là ngày tháng chọn Kiểu_biến là Date

- Nếu biến chỉ có một trong 2 giá trị chọn kiểu biến là Boolean

- Có thể không ðặt kiểu biến với những trýờng hợp khác.

  Tất nhiên không chỉ có từng ấy kiểu biến! Bạn có thể tham khảo ðể rõ hõn.

Sau ðó tiến hành tính toán và gán giá trị cho các biến, tùy theo từng trýờng hợp mà ghi vào biến kết quả. Có thể dùng cấu trúc phân nhánh nhý IF THEN ELSE SELECT CASE, cấu trúc vòng lặp nhý DO WHILE , FOR NEXT , FOR EACH IN   ðể ðặt ðiều kiện ðýa vào biến cần ghi Kết quả. Không nên dùng thẳng luôn tên hàm dễ gây nhầm lẫn.

Khi thiết kế trong Visual Basic   ngýời ta phải sử dụng dữ liệu kiểu Anh Mỹ. Tức là dùng dấu chấm thay cho dấy phảy.

Hàm TODAY() của EXCEL thì trong Visual Basic ðýợc thay bằng Date.

Hàm MOD, AND, OR của EXCEL thì trong Visual Basic thay bằng phép tính MOD, AND, OR.

Ví dụ: Thiết kê hàm CANHHUYEN ðể tính cạnh huyền khi ðã biết 2 cạnh góc vuông.

(Ta biết công thức Pitago (cạnh huyền)^2 = (Cạnh 1)^2 + (Cạnh 2)^2)

FUNCTION CANHHUYEN (CANH1, CANH2)

DIM  BINHPHUONG_C_HUYEN, KQ

BINHPHUONG_C_HUYEN = CANH1^2 + CANH2^2

Kq = BINHPHUONG_C_HUYEN ^(1/2)

CANHHUYEN = Kq

END FUNCTION

Function THAMNIEN (NgayTL as Date) as Integer

Thiết kế một hàm có tên là HH sao cho khi viết = HH (E5; F5) 

(E5 là tên ô ghi ngày nhập; F5 là tên ô ghi số lýợng)

- Nếu số lýợng > 1000 thì sau 100 ngày kể từ ngày nhập sẽ hết hạn.

- Nếu số lýợng ( 1000 thì sau 80 ngày kể từ ngày nhập sẽ hết hạn.

- Nếu ðã hết hạn ðýa ra"Ðã hết hạn"

 

Function HH(Ngaynhap, Soluong) As String

Dim Kq

Dim Songay

Songay = Date - Ngaynhap

IF ((Soluong>100)AND(Songay>100))OR((Soluong<=1000)AND(Songay>80)) THEN

Kq = "Hết hạn"

ELSE

Kq = ""

END IF

HH = Kq

END FUNCTION.

 

Thiết kế một hàm có tên là THU sao cho khi sử dụng ta chỉ cần ðánh =THU(Tên ô dữ liệu kiểu ngày) sẽ nhận ðýợc ðó là ngày thứ mấy. Ta áp dụng hàm WEEKDAY(Ngay) ðể biết ðýợc thứ mấy thông qua code ngày.

 


Option Explicit

Function Thu (ngay As Date) As String

Dim kq, CodeNgay

CodeNgay = WeekDay(ngay)

Select Case CodeNgay

Case 1

kq = "Chủ nhật"

Case 2

kq = "Thứ hai"

Case 3

kq = "Thứ ba"

Case 4

kq = "Thứ tý"

Case 5

kq = "Thứ nãm"

Case 6

kq = "Thứ sáu"

Case 7

kq = "Thứ bảy"

Case Else

kq = "Sai ngày"

End Select

Thu = kq

End Function

3. Vòng lặp FOR EACH OHienTai IN VungO

- Dùng ðể lặp lại một phép tính hoặc một thao tác ðối với từng ô một trong Vùng ô.

Lệnh hoặc phép tính chỉ viết cho 1 ô, nhýng lặp lại týõng tự ðối với toàn bộ các ô trong vùng.

Cú pháp:

FOR  EACH OHienTai IN VungO (Vùng mà cần lặp lại các lệnh dýới cho tất cả các ô)

Các lệnh hoặc phép tính ðối với ô hiện tại

NEXT

Ví dụ: Thiết kế hàm TONGBP (VungDULIEU) biết rằng VungDULIEU là tên vùng có các ô số cần cộng tổng bình phýõng của tất cả các ô.

FUNCTION TONGBP (VDL)

Dim Kq, OHienTai

FOR EACH OHienTai IN VDL

Kq = Kq + OHienTai^2

NEXT

TONGBP = Kq

END FUNCTION

- Muốn kiểm tra các thuộc tính dữ liệu của ô hiện tại ta thýờng dùng ðể kiểm tra thuộc tính Font hoặc thuộc tính nền bằng cách sử dụng công thức:

OHienTai.Font. Thuộc_tính (Lấy thuộc tính Font chữ của ô hiện tại)

Thuộc tính FONT  gồm có:

OHienTai.Font.Name: Phông chữ ðang sử dụng của ô hiện tại.

OHienTai.Font.Size: Cỡ chữ ðang sử dụng của ô hiện tại

OHienTai.Font.Color: Màu chữ của ô hiện tại.

OHienTai.Interiror. Color (Lấy thuộc tính màu nền của ô hiện tại)

Ví dụ: Thiết lập hàm mới có tên là CONGTHEOMAU(VungDULIEU, $C$3)

(Nếu ô nào là ô số trong vùng có màu trùng với màu của ô mẫu C3 thì cộng lại vào ðýa kết quả tổng).

FUNCTION CONGTHEOMAU (VungDL, OMau)

Dim OHienTai, KetQua, MauChuan, MauHienTai

MauChuan = OMau.Interiror.Color

FOR EACH OHienTai IN VUNGDL

MauHienTai  = OHienTai.Interiror.Color

IF ISNumeric (OHienTai) AND (MauHienTai = MauChuan) THEN

KetQua = KetQua + OHienTai

NEXT

CONGTHEOMAU = KetQua

END FUNCTION
